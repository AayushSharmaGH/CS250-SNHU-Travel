# CS 250 - SNHU Travel Booking Application

## Project Overview
This repository contains all the work I completed for **CS 250: Software Development Life Cycle** at Southern New Hampshire University. 

The project involved developing a travel booking application for **SNHU Travel** using **Scrum-Agile** methodology. Throughout the course, I took on multiple roles including Developer, Tester, Product Owner, and Scrum Master.

---

## Repository Contents
- **Sprint Review and Retrospective** – Final written reflection (Word document)
- **Agile Presentation** – PowerPoint slides comparing Waterfall and Agile
- **SlideShow Control** – Java application developed in Module 5 (wellness/detox theme)
- **Weekly Journals & Reflections** – Work from all modules
- **Discussion Posts** – Group collaboration as Scrum Master

## Reflection Questions

**1. What was the most valuable lesson you learned while working in an Agile environment?**  
Adapting quickly to changing requirements (wellness pivot in Module 5) taught me the real power of Agile flexibility.

**2. How did taking on different Scrum roles help you understand the development process?**  
Experiencing Developer, Tester, Product Owner, and Scrum Master roles gave me a full understanding of how each role contributes to project success.

**3. What challenges did you face during the project, and how did Agile help overcome them?**  
The major change in project direction was handled smoothly thanks to short sprints and the ability to reprioritize the backlog.

**4. Would you recommend Agile/Scrum for future projects? Why?**  
Yes. Agile provided better collaboration, faster feedback, and flexibility compared to traditional Waterfall methods.
